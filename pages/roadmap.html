14/09/2015: We are very proud to announce the release of wallabag 2.0.0-alpha Please read our blog post.

Roadmap

wallabag v2.0.0-alpha

  • [core] login / logout
  • [core] RESTful API
  • [core] save a new link
  • [core] new design
  • [core] mark an article as read, as favorite, delete it
  • [core] change configuration (theme, items per page, language)
  • [core] change user information (username, name, email, password)
  • [core] create new user
  • [core] tags
  • [core] edit entry (title, visibility)
  • [core] filters on domain name, reading time, creation date, etc.

wallabag v2.0.0

  • [wallabag.org] new website
  • [doc] user documentation #1359
  • [filters] autocomplete on domain name #1380
  • [core] remove tags from an entry #1377
  • [core] assign multiple tags at once #1376
  • [core] public registration
  • [core] [API] oAuth implementation #795
  • [core] admin interface #1346
  • [deployment] Travis CI job to deploy Framabag with Capistrano
  • [core] entries export (PDF, json, ePub, mobi, txt, html)
  • [core] undo action after deletion
  • [core] pagination more intuitive #1329
  • [smartphone] ionic app
  • [deployment] launch test suite using a dedicated ENV variable to define the current theme #1373
  • [quality] fix scrutinizer issues #1360
  • [framabag] maintenance mode #1351

wallabag v2.1.0

  • [core] sync read position between web app and clients #1336
  • [API] implement filters #1313
  • [core] store entry language #1333
  • [filters] entry language
  • [core] share entries between users
  • [core] delete my account

wallabag v2.2.0

  • [core] plugin system
  • [plugin] export data on cloud service
  • [core] edit site_config in my wallabag account
